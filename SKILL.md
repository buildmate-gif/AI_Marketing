## ルーティングロジック

ユーザーが `/market <コマンド>` を実行した場合、適切なサブスキルにルーティングします。

### 完全マーケティング監査（`/market audit <url>`）
これはフラッグシップコマンドです。**5つの並列サブエージェント**を起動してウェブサイトを同時分析します：

1. **market-content** エージェント → コンテンツ品質・メッセージング・コピーの効果
2. **market-conversion** エージェント → CRO・ファネル・ランディングページ・登録フロー
3. **market-competitive** エージェント → 競合ポジショニング・市場環境
4. **market-technical** エージェント → テクニカルSEO・サイト構造・ページ速度
5. **market-strategy** エージェント → 全体戦略・価格設定・成長機会

**スコアリング方式（マーケティングスコア 0-100）：**
| カテゴリ | ウェイト | 評価内容 |
|---------|--------|---------|
| コンテンツ・メッセージング | 25% | コピー品質・価値提案・明確さ・説得力 |
| コンバージョン最適化 | 20% | CTA・フォーム・摩擦・社会的証明・緊急性 |
| SEO・発見可能性 | 20% | オンページSEO・テクニカルSEO・コンテンツ構造 |
| 競合ポジショニング | 15% | 差別化・市場認知・代替サービスページ |
| ブランド・信頼性 | 10% | ブランド一貫性・信頼シグナル・社会的証明 |
| 成長・戦略 | 10% | 価格設定・紹介・リテンション・拡張機会 |

**総合マーケティングスコア** = 全6カテゴリの加重平均

### クイックスナップショット（`/market quick <url>`）
60秒の高速評価。サブエージェントは起動しません。代わりに：
1. WebFetchでホームページを取得
2. 評価項目：ヘッドラインの明確さ・CTAの強度・価値提案・信頼シグナル・モバイル対応
3. トップ3の改善点と上位3つの問題点を含むスコアカードを出力
4. 出力は30行以内に収める

### 個別コマンド
その他のコマンド（`/market copy`、`/market emails` など）は `skills/market-<command>/SKILL.md` の対応するサブスキルにルーティングします。

## ビジネスタイプの検出

分析実行前に、必ずビジネスタイプを検出してください：
- **SaaS・ソフトウェア** → 重点：トライアル→有料転換・オンボーディング・機能ページ・料金プラン
- **EC・通販** → 重点：商品ページ・カート放棄・アップセル・レビュー
- **エージェンシー・サービス業** → 重点：事例・ポートフォリオ・お問い合わせフォーム・信頼シグナル
- **ローカルビジネス** → 重点：Googleビジネスプロフィール・ローカルSEO・口コミ・アクセス情報
- **クリエイター・講座** → 重点：リードマグネット・メール登録・推薦文・コミュニティ
- **マーケットプレイス** → 重点：双方向メッセージング・需給バランス・信頼メカニズム
- **建設・建築業** → 重点：施工実績・建設業許可・有資格者・お問い合わせ・来場予約
- **製造業・BtoB** → 重点：カタログ請求・展示会申込・見積依頼・取引実績・ISO認証

## 出力基準

すべての出力は以下のルールに従ってください：
1. **実行可能性を優先** — すべての推奨事項は具体的に実施できる内容にする
2. **優先順位をつける** — 必ずインパクト順（高・中・低）でランク付けする
3. **収益に結びつける** — すべての提案をビジネス成果に関連付ける
4. **具体例を示す** — アドバイスだけでなく、改善前・改善後のコピー例を含める
5. **クライアントに提出できる品質** — レポートは編集なしにクライアントへ提出できる水準にする

## ファイル出力

詳細な出力はカレントディレクトリのMarkdownファイルに保存します：
- わかりやすいファイル名を使用：`MARKETING-AUDIT.md`、`COMPETITOR-REPORT.md` など
- ファイル先頭にURL・日付・総合スコアを記載
- 見出しと表で明確に構造化する
- クライアント向けレポートにはエグゼクティブサマリーを含める

## スキル間の連携

多くのスキルは連携して機能します：
- `/market audit` → 全サブエージェントを呼び出し総合分析を生成
- `/market proposal` → 監査結果が存在する場合はそれを参照
- `/market report` と `/market report-pdf` → 利用可能な全分析データを集約
- `/market copy` → 先に `/market brand` を実行するとブランドボイスを活用できる
- `/market emails` → `/market funnel` の分析結果があれば活用する
