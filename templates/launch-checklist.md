# サービス・製品ローンチチェックリスト

## ローンチ8週間前

### 戦略
- [ ] ローンチ目標を設定する（売上目標・獲得ユーザー数・認知度目標）
- [ ] ローンチのターゲット顧客層を特定する
- [ ] ローンチの方法を決める（メール・SNS・広告・パートナー・コミュニティ等）
- [ ] ローンチ日時を確定する
- [ ] 成功指標と計測方法を定義する

### サービス・製品
- [ ] ローンチバージョンの機能・内容が完成している
- [ ] ランディングページが構築・最適化されている
- [ ] 価格・料金が確定している
- [ ] 申し込み・登録フローがテスト済みである
- [ ] よくある質問（FAQ）・サポートページが作成されている

---

## ローンチ4週間前

### コンテンツ
- [ ] ローンチ用ブログ記事・告知文を書く
- [ ] メールシーケンスを作成する（予告・当日・フォローアップ・締め切り）
- [ ] ローンチ週のSNS投稿を用意する（全プラットフォーム）
- [ ] サービス・製品の紹介動画を制作する
- [ ] ローンチ用の画像・ビジュアルを作成する
- [ ] プレスリリースや取材ピッチを書く（必要な場合）

### 見込み顧客の獲得
- [ ] 事前登録ページを設置する
- [ ] SNSでの予告投稿を開始する
- [ ] 10〜20名のモニター・テストユーザーに声をかける
- [ ] 提携候補・紹介者に連絡を取る
- [ ] ローンチを告知してもらえそうなインフルエンサー・発信者を5〜10名特定する

### 技術的な準備
- [ ] ローンチページのアナリティクストラッキングを設定する
- [ ] 申し込み・決済フローをエンドツーエンドでテストする
- [ ] メール自動配信を設定する
- [ ] ローンチ用リンクにUTMパラメータを設定する
- [ ] インフラの負荷テストを行う（必要な場合）

---

## ローンチ1週間前

### 最終確認
- [ ] 予告メールをリスト全体に送信する
- [ ] SNS投稿を予約投稿としてスケジュールする
- [ ] チームメンバーにローンチ計画を共有する
- [ ] よくある質問への回答を事前に用意する
- [ ] ローンチ当日の情報共有体制を整える（Slackチャンネル・ダッシュボード等）
- [ ] 全リンク・フォーム・申し込みフローを再度テストする

### コミュニティ
- [ ] ローンチ当日のSNS投稿文を下書きする
- [ ] 予想される質問への回答を準備する
- [ ] 応援してくれる知人・フォロワーに当日の拡散をお願いする

---

## ローンチ当日

### 午前中
- [ ] ローンチメールをリスト全体に送信する
- [ ] 全SNSチャンネルでローンチ告知を投稿する
- [ ] パートナー・紹介者・応援者に連絡する
- [ ] アナリティクスダッシュボードを監視する

### 1日を通して
- [ ] コメント・質問に1時間以内に返信する
- [ ] 最初の申し込みやユーザーの声をSNSでシェアする
- [ ] 最初のお客様に個別にお礼を伝える
- [ ] バグや技術的な問題がないか監視する
- [ ] 進捗・マイルストーンを随時投稿する

### 夕方
- [ ] 社会的証明メールを送信する（初日の成果・お客様の声）
- [ ] 応援してくれたコミュニティへの感謝を伝える
- [ ] 気づきを記録しておく（記憶が鮮明なうちに）

---

## ローンチ後（第1〜2週）

### フォローアップ
- [ ] フォローアップメールを送信する（実績紹介・FAQ・締め切り案内）
- [ ] 特に力になってくれた方々に個別で連絡する
- [ ] お客様の声・推薦文を集めて公開する
- [ ] 最も良い初期成果から事例を作成する
- [ ] 期間限定オファーを締め切る（設定している場合）

### 分析・振り返り
- [ ] ローンチ指標を目標と照合する
- [ ] 最も効果的だったチャネルを特定する
- [ ] うまくいったこと・いかなかったことを記録する
- [ ] ローンチ後のコンテンツ戦略を計画する
- [ ] ローンチデータをもとに30日・90日の目標を設定する

---

## 計測すべき主要指標
| 指標 | 計測方法 |
|------|---------|
| 申し込み・成約数 | 決済システム ＋ アナリティクス |
| 売上 | 決済ダッシュボード |
| CVR（コンバージョン率） | 訪問者 → 申し込み |
| メール開封率 | メール配信ツール |
| SNSインプレッション | 各プラットフォームのアナリティクス |
| お客様のフィードバック | 問い合わせ受信箱 ＋ SNSのメンション |
| 紹介経由の申し込み | UTMトラッキング |
